# Amazon S3

## Visão Geral

O **Amazon Simple Storage Service (Amazon S3)** é o serviço de armazenamento de objetos utilizado pela aplicação para armazenar arquivos permanentes, backups do banco de dados, documentos enviados pelos usuários e certificados emitidos pela instituição de ensino.

Além do armazenamento seguro e altamente disponível, o Amazon S3 permite aplicar políticas de ciclo de vida (Lifecycle), versionamento e diferentes classes de armazenamento, reduzindo custos sem comprometer a durabilidade dos dados.

Além do armazenamento seguro e altamente disponível, o Amazon S3 permite aplicar políticas de ciclo de vida (Lifecycle) e diferentes classes de armazenamento para otimização de custos. Neste projeto, os objetos são inicialmente armazenados utilizando a classe **S3 Intelligent-Tiering**, permitindo que a AWS mova automaticamente os arquivos entre camadas de acesso conforme o padrão de utilização. Para os certificados emitidos pela instituição, é aplicada uma política adicional de Lifecycle que os transfere para a classe **S3 Glacier Deep Archive**, onde permanecem armazenados durante o período de retenção de cinco anos.


---

# Objetivos

A utilização do Amazon S3 possui os seguintes objetivos:

* Armazenar arquivos enviados pelos usuários;
* Disponibilizar certificados digitais para download;
* Armazenar backups lógicos do banco PostgreSQL;
* Reduzir custos através de políticas de tierização;
* Disponibilizar conteúdo estático para distribuição pelo CloudFront;
* Armazenar logs de monitoramento da infraestrutura;
* Garantir alta durabilidade e disponibilidade dos arquivos.

---

# Utilização nas Arquiteturas do Projeto

## Proposta Low-Cost

Na arquitetura Low-Cost, o Amazon S3 é utilizado como armazenamento central de arquivos da aplicação.

São armazenados:

* Backups lógicos do PostgreSQL;
* Históricos escolares enviados pelos usuários;
* Certificados emitidos pela instituição;
* Demais arquivos enviados pela aplicação.

Os certificados permanecem disponíveis para consulta durante cinco anos.

Após um período definido pela política de ciclo de vida, os certificados são automaticamente migrados para a classe **Amazon S3 Glacier Deep Archive**, reduzindo significativamente os custos de armazenamento de longo prazo.

---

## Proposta de Maior Investimento

Na arquitetura de maior investimento, o Amazon S3 mantém todas as funções da proposta Low-Cost e incorpora novos serviços integrados.

Além do armazenamento dos arquivos da aplicação, o S3 passa a ser utilizado para:

* Armazenamento de logs exportados do Amazon CloudWatch;
* Origem de conteúdo estático para o Amazon CloudFront;
* Distribuição otimizada de imagens e arquivos estáticos através da CDN.

Essa integração melhora o desempenho da aplicação e reduz a latência para usuários distribuídos geograficamente.

---

# Arquitetura de Armazenamento

O ambiente utiliza diferentes categorias de arquivos.

| Tipo de Arquivo              | Finalidade                                           |
| ---------------------------- | ---------------------------------------------------- |
| Backup PostgreSQL            | Recuperação do banco de dados                        |
| Histórico Escolar            | Upload realizado pelos usuários                      |
| Certificados                 | Download pelos alunos                                |
| Logs CloudWatch              | Monitoramento da infraestrutura (Maior Investimento) |
| Imagens e arquivos estáticos | Distribuição via CloudFront (Maior Investimento)     |

# Política de Armazenamento

Todos os objetos armazenados pela aplicação utilizam inicialmente a classe **Amazon S3 Intelligent-Tiering**, permitindo a otimização automática dos custos de armazenamento sem impacto para a aplicação.

Essa classe monitora continuamente o padrão de acesso aos objetos e realiza a movimentação automática entre suas camadas internas conforme a frequência de utilização.

## Backups

Os arquivos de backup do PostgreSQL permanecem armazenados conforme a política definida em **backup.md**, utilizando inicialmente o armazenamento em **S3 Intelligent-Tiering**.

---

## Históricos Escolares

Os históricos escolares enviados pelos usuários permanecem armazenados em **S3 Intelligent-Tiering**, garantindo disponibilidade imediata para acesso quando necessário, enquanto os custos são automaticamente otimizados de acordo com o padrão de utilização.

---

## Certificados

Os certificados emitidos pela instituição também são armazenados inicialmente em **S3 Intelligent-Tiering**.

Como esses documentos costumam ser acessados com menor frequência após sua emissão, é aplicada uma política de **Lifecycle** para migração automática ao **Amazon S3 Glacier Deep Archive**, onde permanecem armazenados durante o período de retenção de cinco anos.

Essa estratégia reduz significativamente os custos de armazenamento de longo prazo, preservando a elevada durabilidade oferecida pelo Amazon S3.

Fluxo simplificado:

```text
Emissão do Certificado
          │
          ▼
 Amazon S3 Intelligent-Tiering
          │
          ▼
   Período de utilização
          │
          ▼
   Lifecycle Policy
          │
          ▼
 Amazon S3 Glacier Deep Archive
          │
          ▼
 Retenção de 5 anos
```

# CloudFront (Maior Investimento)

Na proposta de maior investimento, o Amazon S3 também atua como origem de conteúdo estático.

Fluxo simplificado:

```text
Usuário
    │
    ▼
CloudFront
    │
    ▼
Imagem disponível em cache?
       │
 ┌─────┴─────┐
 │           │
Sim         Não
 │           │
 ▼           ▼
Entrega     Amazon S3
Imediata       │
               ▼
         Armazena em Cache
               │
               ▼
          Entrega ao Usuário
```

Essa estratégia reduz a latência de acesso às imagens e diminui a quantidade de requisições diretamente ao bucket S3.

---

# Armazenamento de Logs

Na proposta de maior investimento, os logs exportados do Amazon CloudWatch são armazenados no Amazon S3 para fins de auditoria, monitoramento e retenção histórica.

Fluxo simplificado:

```text
Aplicação
      │
      ▼
Amazon CloudWatch
      │
      ▼
Exportação
      │
      ▼
Amazon S3
      │
      ▼
Armazenamento dos Logs
```

Essa abordagem facilita análises posteriores e reduz custos de retenção de logs em longo prazo.

# Benefícios

## Proposta Low-Cost

* Armazenamento altamente durável;
* Backup dos dados da aplicação;
* Upload de documentos pelos usuários;
* Download de certificados;
* Redução de custos através do Glacier Deep Archive.

---

## Proposta de Maior Investimento

* Todos os benefícios da proposta Low-Cost;
* Distribuição global de conteúdo utilizando CloudFront;
* Cache de imagens e arquivos estáticos;
* Menor latência para os usuários;
* Armazenamento centralizado dos logs do CloudWatch;
* Melhor desempenho geral da aplicação.

---

# Considerações Finais

O Amazon S3 desempenha um papel fundamental nas duas arquiteturas propostas, oferecendo armazenamento seguro, escalável e altamente disponível para os arquivos da aplicação.

Na proposta Low-Cost, o serviço concentra-se no armazenamento de documentos, certificados e backups do banco de dados, utilizando políticas de ciclo de vida para reduzir custos.

Na proposta de maior investimento, o Amazon S3 assume funções adicionais de distribuição de conteúdo estático por meio do Amazon CloudFront e armazenamento de logs do Amazon CloudWatch, contribuindo para uma arquitetura mais eficiente, resiliente e alinhada às boas práticas de ambientes corporativos.
